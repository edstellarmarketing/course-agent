Card density is deliberate. One suggestion per scroll-height means reviewers can't speed-click without seeing the whole context — title, rationale, price basis, sources, and the closest existing course. Most rejections happen because the candidate IS already in inventory; surfacing that side-by-side prevents wasted decisions.

Reject with Structured Reason

/suggestions/today · modal

The single most important interaction in the system. Reviewers must select at least one tag from the structured taxonomy — free text alone produces a noisy signal the agent can't learn from. The tags become the agent's negative memory for the next 90 days.

Multi-select tags reflect reality — a rejection is often "wrong category AND too niche." The free-text note becomes a re-prompt hint if the same topic later resurfaces in a slightly different shape. Tags are enforced server-side; the agent's negative-memory blocklist is built directly from this table.
